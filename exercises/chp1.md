1.  Q1
  - (a) 디바이스의 센서와 액추에이터의 역할을 비교하여 설명하세요.
      - 센서에서는 정보를 읽어 메시지 브로커로 전송한다.
      - 브로커에서 수신된 정보에 따라 액추에이터가 작동한다.
  - (b) 디바이스와 대시보드에서 동일한 프로그래밍 언어를 사용하지 않는 이유를 설명하세요.
      - 디바이스에서는 메모리나 배터리 용량이 제한되어 상대적인 저수준 언어가 필요한 반면
    대시보드에서는 디바이스에서 받는 정보와 데이터에 대해 표시하고 처리하는데 중점을 두고 있어 개발 편의성이 높은 상대적 고수준 언어를 주로 사용한다.
  - (c) 브로커와 같은 정보 전달 전담 서버의 이점을 설명하세요.
      - 기기의 리소스와 전력을 절감할 수 있으며 비동기 처리가 가능해 속도 상승과 안정성을 기대할 수 있다. 기기가 브로커와 통신만 하면 되므로 기기 확장이 쉽고 외부 인터넷         과 접점이 줄어 보안에 유리하다

  - (c )  Mosquitto와 Node_RED에 대해 설명하고, 실행에 적합한 컴퓨팅 환경을 제시하십시오
      - Mosquitto는 MQTT(Message Queuing Telemetry Transport)프로토콜을 사용한 메시지 브로커로 센서에서 정보를 받고, 송출하는 역할을 맡는다.
      - Node-Red는 플로우 기반의 로우코드 프로그래밍 도구로 데이터를 가공하고 대시보드를 구축하거나 장치를 제어한다.
      - 둘 모두 가벼운 프로그램으로 낮은 사양의 환경에서 강점을 보인다. 임베디드 시스템, 미니 PC등의 하드웨어와 리눅스 기반의 OS가 해당된다.
